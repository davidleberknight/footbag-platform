# Freestyle Footbag Glossary (v3)

**Purpose**
This glossary explains freestyle footbag terminology: run quality, trick components, style, and community vocabulary.

**Important:**
This is an **explanatory layer only**.
Canonical definitions (ADD values, trick structure, modifier rules) live in the freestyle dictionary system (`freestyle_tricks`, `freestyle_trick_modifiers`).

---

## 1. ADD System & Run Quality

These terms describe the **difficulty of a run**, not individual tricks.

| Term      | Meaning                         |
| --------- | ------------------------------- |
| Tiltless  | All tricks in a run are ≥ 2 ADD |
| Guiltless | All tricks are ≥ 3 ADD          |
| Tripless  | All tricks are ≥ 4 ADD          |
| Fearless  | All tricks are ≥ 5 ADD          |
| Beastly   | All tricks are ≥ 6 ADD          |
| Godly     | All tricks are ≥ 7 ADD          |

Additional:

| Term    | Meaning                         |
| ------- | ------------------------------- |
| Genuine | Guiltless excluding BOP tricks  |
| BOP     | Butterfly, Osis, Paradox Mirage |

👉 These describe **run quality**, not trick definitions.

## 2. Core Trick Concepts

### ADD (Additional Degree of Difficulty)

A scoring system assigning difficulty values to tricks based on their components.

---

### Dexterity (Dex)

A motion where the foot circles around the bag.

**Example:**

* Mirage = one dex
* Double Around the World = two dexes

---

### Set

The action used to position the bag before a trick.

* **Uptime** — before the peak
* **Midtime (hangtime)** — at the peak
* **Downtime** — after the peak

👉 Modern usage: *set = uptime + midtime components*

---

### Stall / Delay

Controlling the bag on a surface.

**Examples:**

* Toe stall
* Clipper stall
* Inside stall

👉 Most stalls are **1 ADD**

---

### X-Dex

A dexterity performed in a crossed-body position.

**Example:**

* Atom Smasher uses an X-dex

---

## 3. How Tricks Are Built (Conceptual)

A trick is typically composed of:

```text
set → modifiers → base → catch (stall or kick)
```

* **Base trick** = core motion (e.g., mirage, whirl)
* **Modifiers** = how the trick is altered (e.g., paradox, spinning)
* **Set** = how the trick begins

---

### Example

```text
Paradox Whirl = whirl (base) + paradox (modifier)
```

```text
Ripwalk = butterfly (base) + stepping (modifier)
```

👉 The glossary explains terms; the dictionary defines exact structure.

## 5. Runs, Combos, and Style

These describe **how tricks are used**, not how they are defined.

---

### Run

A continuous sequence of tricks.

---

### Shuffle

Alternating sides rhythmically.

---

### Link / Transition

The connection between tricks.

---

### Connector Trick

A trick commonly used to maintain flow.

**Example:**

* Whirl-family tricks often act as connectors

---

### Dropless

A run without drops.

---

### Density

Average ADD per trick in a run.

---

### Shred Circle

A group format where players take turns performing runs.

---

## 6. Play Quality & Style Terms

These describe execution quality rather than structure.

| Term   | Meaning                                 |
| ------ | --------------------------------------- |
| Clean  | Fully controlled, clearly executed      |
| Sloppy | Poorly controlled execution             |
| Thin   | A dex that barely clears the bag        |
| Slurry | Dex completed too early (not clean)     |
| Froggy | Body motion begins before set completes |

👉 These do not affect canonical trick definitions.

---

## 7. Notation (Jobs Notation Overview)

A symbolic system describing trick structure.

### Example

```text
(clipper) > whirl > butterfly
```

This represents:

* starting surface
* intermediate components
* final catch

👉 In this project, notation is stored as text and not yet fully parsed.

## 9. Competitive & Format Terms

These describe event formats and play contexts.

* **Freestyle Routine** — choreographed performance to music
* **Shred Contest (Shred:30)** — technical scoring format
* **Big Trick / Sick, Big, Best** — single or combo highlight formats
* **Circle Contest** — turn-based elimination format
* **Most Rippin’ Run** — last-player-standing format
