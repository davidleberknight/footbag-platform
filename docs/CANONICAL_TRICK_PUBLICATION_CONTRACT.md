# Canonical Trick Publication Contract -- Freestyle Footbag

A trick must not be promoted to accepted canonical status unless it is structurally legible across the freestyle symbolic system.

## Minimum publication requirements

### 1. Symbolic representation

Every accepted trick must have at least one of:

- curator-authored notation
- curated equivalence reading
- structural / base-lineage rendering
- explicit pending-curation state

Silent semantic absence is not acceptable for non-core tricks.

### 2. Structural composition

Every accepted trick must expose:

- core/base relationship

OR

- explicit irreducible-core status

Intermediate operators may remain compressed but must be explainable somewhere authoritative.

### 3. Discoverability across domains

Every accepted trick must appear in multiple semantic/browse domains where applicable:

- family
- component
- topology
- ADD
- glossary/operator linkage
- media/records
- equivalence chains

A trick should never exist as an isolated canonical row.

### 4. Alias/equivalence governance

Known aliases, folk names, PassBack names, and equivalent readings must be:

- mapped
- classified
- rejected
- or explicitly deferred

Never silently ignored.

### 5. Honest incompleteness

Missing decomposition data is a curation gap, not proof of atomicity.

Pending states must render honestly.

### 6. No fabricated structure

Do not:

- infer deep decomposition from names
- auto-generate recursive chains
- fabricate operator lineage
- normalize away community vocabulary

Curator authority remains primary.

## Arithmetic validity is not structural certainty

A trick's ADD arithmetic and its structural decomposition are independent gates. The math closing (the ADD value derives cleanly, and any source divergence on the number resolves under the outlier rules) does not by itself make a trick publishable.

Structural certainty is a separate requirement. A trick can carry a correct, agreed ADD and still fail publication because its structure is unreconciled: which operators compose it, which base it lands on, or which of several competing readings is canonical. When the number is settled but the structure is contested, the trick is held, not published. Holding such a trick is the contract operating as designed, not a coverage gap.

This distinction governs requirements 4 (alias and equivalence governance), 5 (honest incompleteness), and 6 (no fabricated structure): a clean ADD never licenses asserting a structure the sources do not agree on.

## Core principle

A canonical trick is a structurally legible symbolic object, not merely a database row.
