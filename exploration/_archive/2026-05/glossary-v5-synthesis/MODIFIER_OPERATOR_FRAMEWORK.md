# MODIFIER_OPERATOR_FRAMEWORK

**Project:** GLOSSARY-V5-SYNTHESIS — Task D
**Scope:** Draft prose for §6 of the V5 glossary architecture — modifiers and operators. Modifiers framed as **transformational operations** that take a base trick and produce a new one. Five operator classes (body / set / timing / compositional / directional). Progressive teaching: simple operators first; compound revelations earned. Body mechanics first; notation supports.
**Constraints:** No invented tokens. Compound examples cited from canonical IFPA structure where available, otherwise flagged. No ADD-rule changes.

---

## How to read this section

A **modifier** is not a glossary word to memorize. It's an *operator* — a function that takes a base trick and produces a compound trick by transforming one window of the trick's timing clock.

Once you start hearing modifiers this way, names like *Ripwalk = Stepping Butterfly* stop looking like vocabulary lists and start looking like structural sentences. Stepping is the operator; Butterfly is the operand; Ripwalk is the result. The operator's mechanic — what it does to the body — is the same whenever it appears, regardless of which base it's transforming.

This section introduces the operators in roughly the order a learner can absorb them. The simplest single-mechanic operators come first. The compound operators — the ones that stack on top of simpler ones — come later. By the end you'll be able to read compound names as compositions instead of opaque vocabulary.

A note before we begin: **compound revelations are earned**. We won't lead with "Mobius = Spinning Torque" until you've met spinning, until you've met torque, and until both have their own pedagogical footing. The point of this section isn't to dump compositions; it's to make them visible *when the body knowledge is ready for them*.

## §6.1 The operator framing

Picture a trick's timing clock from the primer:

```
contact → uptime → midtime → downtime → catch
```

Every modifier transforms **one or more windows** of that clock. When you apply Stepping to a Butterfly, the *uptime* window changes — it now carries an extra foot-relocation. The Butterfly's *downtime dex* is still there; the *catch* is still there. Stepping fills the uptime that was previously empty.

When you apply Spinning to a Whirl, the *body action during the dex* changes — the whole body rotates 360° while the dex happens. The Whirl's leggy-in dex is still there; the catch is still there. Spinning adds a rotational layer to the existing motion.

This is the core mental model. **Modifiers are operators on windows of the clock.** Identify the window the modifier transforms, and you can reason about what the compound will feel like.

We can group operators into five classes by *which window they transform*:

| Class | What it transforms | Examples |
|---|---|---|
| **Body modifiers** | The body's posture during a window | spinning, gyro, paradox, ducking, diving, weaving, zulu, symposium |
| **Set modifiers** | The uptime window's content | stepping, pixie, fairy, atomic, quantum, nuclear, furious, blurry |
| **Timing modifiers** | Which window holds which component | symposium, symple, muted, alpine, crispy |
| **Compositional operators** | The relationships between components | ss / op (sides), in / out (directions), fullness |
| **Directional modifiers** | The rotational direction of a body action | inspin, in- prefix |

The classes overlap at the edges — symposium appears in both body-modifier and timing-modifier lists because it modifies both the support leg's behavior AND the timing window the dex occupies. The boundaries between classes are pedagogical, not crisp.

We'll meet them in order: directional and compositional operators first (you've already met side and direction in the primer), then set modifiers (because they're often the simplest mechanically), then body modifiers (the heart of the freestyle vocabulary), then timing modifiers (which often layer on top), then compound operators that stack multiple at once.

## §6.2 Compositional and directional operators (review from the primer)

You've already met these. They're not modifiers in the usual sense — they don't transform a base trick into a different named trick. They modify *how a trick is described*.

- **ss / op** — same-side or opposite-side, relative to the previous component. Already covered in the primer.
- **in / out** — the direction of a dex's circling motion. Already covered in the primer.
- **inspin** / **in-** prefix — flips the rotational direction of a body modifier. A spin defaults to backspin (back passes the bag first); inspin reverses it (chest passes the bag first). The `in-` prefix can attach to "spinning," "gyro," or even full trick names (`in-vortex` for an inspin-gyro-drifter).

These aren't operators in the "transform the base" sense; they're refinements. We list them here because they're part of the modifier vocabulary, but they're best learned by encountering them in worked examples.

## §6.3 The simplest set modifiers — stepping

**What it does:** Stepping transforms the uptime window. The kicking leg performs a foot-relocation — a stepping motion — *while the bag is still rising*. The result is that the leg ends in a different position than where it started before the downtime dex begins.

**Body mechanics:** The set foot relocates mid-air. There's no dex in the stepping motion — it's a *transport*, not a circling. The leg's job in stepping is to move itself; the bag keeps rising on its own. By the time the bag begins its descent, the leg is in a new position, ready for the downtime dex.

**Transformation rule:** `Stepping + [base] = compound with stepping in the uptime`. The base's downtime dex and catch are unchanged.

**Worked example: Ripwalk = Stepping Butterfly.** The Butterfly base is `[set] > out dex > ss clipper` — hippy out dex, same-side clipper catch. Adding Stepping to the uptime gives you Ripwalk: the set foot relocates during uptime, then the hippy out dex happens during downtime, then the same-side clipper catch finishes. Same body parts; same catch; new uptime motion inserted.

You can hear the compound in the name: *Stepping* (the operator) + *Butterfly* (the operand) = *Ripwalk* (the result, given a folk name for cultural reasons). The folk name "Ripwalk" is a single-token compression of the two-word compositional reading "Stepping Butterfly." Both refer to the same trick.

**ADD contribution:** +1 ADD (stepping adds one degree of difficulty as a body component).

**Relatives:** Pixie (a more compressed uptime set treatment); Atomic (a more demanding uptime set with x-dex character); Quantum (the compressed form of Atomic).

## §6.4 The pixie set modifier

**What it does:** Pixie is a **compressed pre-base uptime set**. Like stepping, it fills the uptime window. Unlike stepping, the pixie motion is shorter — a quick, compact set with less travel.

**Body mechanics:** Where stepping has the leg take a deliberate relocation, pixie compresses the set into a brief, tight motion. The leg moves, but the motion is small. The leg is essentially "ready faster" for the downtime dex.

**Transformation rule:** `Pixie + [base] = compound with pixie in the uptime`. Same shape as Stepping, different uptime treatment.

**Worked example: Dimwalk = Pixie Butterfly.** Same base (Butterfly) as Ripwalk, but with pixie instead of stepping. Body feel is different: Dimwalk reads as more compact and quick; Ripwalk reads as broader. The compositional structure is identical (`uptime modifier + butterfly base`); the body experience differs because the uptime modifier differs.

**Worked example: Phoenix = Pixie Ducking Butterfly.** Now we're stacking. The Butterfly base is still there; the Pixie operator transforms the uptime; the Ducking operator transforms the midtime (we'll meet ducking shortly). Three operators on one base, each transforming a different window.

If you can hear Ripwalk as "stepping + butterfly," and Dimwalk as "pixie + butterfly," then Phoenix becomes hearable as "pixie + ducking + butterfly" — three modifiers stacked, each filling its own window. That's the compositional reading the language enables.

**ADD contribution:** +1 ADD.

**Relatives:** Stepping (the more extended uptime sibling); Quantum (a compressed form named for its tightness); Atomic (a heavier uptime treatment with cross-body character).

## §6.5 The ducking family — head-motion body modifiers

**What it does:** Ducking transforms the midtime window. The head dips toward the bag, the bag passes around the neck, and the trick continues. The body moves the head; the bag's flight is essentially unchanged.

**Body mechanics:** The head and neck move. The shoulders move along for balance, but the body stays largely where it is. The head dips out of the bag's path, and the bag traces a small arc around the side of the neck. The rest of the body — the kicking leg, the support leg, the hips, the timing of the dex — continues through the trick without modification.

**The head-motion family:** Ducking is one of four siblings, distinguished by the head's direction and the side the bag falls:

| Sibling | Head motion | Bag falls |
|---|---|---|
| **Ducking** | toward the bag | opposite side |
| **Weaving** | toward the bag | same side |
| **Diving** | over and under the bag (full arc) | same side |
| **Zulu** | over and under the bag (full arc) | opposite side |

All four are +1 body modifiers, all four transform the midtime window, all four are conceptually a "head dip / dive" inserted between the trick's other components. The four-way distinction is a 2×2 matrix of head direction × bag-fall side.

**Transformation rule:** `Ducking + [base] = compound with a head dip in the midtime`. The base's uptime and downtime are unchanged. Same applies for weaving / diving / zulu — each transforms the midtime, the others stay in place.

**Worked example: Ducking Butterfly.** Butterfly is `[set] > out dex > ss clipper`. Adding Ducking gives a head dip in midtime: the bag rises, the leg sets, the head dips (the duck) between uptime and downtime, the dex happens, the catch follows. The base trick is preserved; one new motion is inserted.

**Worked example: Phoenix = Pixie Ducking Butterfly.** As we saw in §6.4, three operators stacked: pixie (uptime), ducking (midtime), butterfly (base). The compositional reading is the trick's technical name.

**ADD contribution:** +1 ADD per head-motion modifier.

**Relatives within the family:** Weaving / Diving / Zulu (the other head-motion siblings). Across families: Symposium (a leg-motion body modifier with similar +1 ADD weight, different body region).

## §6.6 The spinning family — full-body rotation

**What it does:** Spinning transforms the *body's posture during a dex window*. The entire body rotates 360° while the dex happens; the bag's flight is preserved; the body simply turns through the dex moment.

**Body mechanics:** Where ducking moves only the head, spinning moves the entire body. The hips drive the rotation; the chest and arms follow; the kicking leg follows along the spin's arc. The dex itself still happens, but it happens *while* the body is rotating. The result reads as one continuous turn of the body, with the dex embedded inside it.

The most important biomechanical point: the spin and the dex happen *together*, not in sequence. If the spin finishes before the dex begins, the result reads as "a kick and a spin" — two separate motions. The trick is named "Spinning [base]" because the rotation and the dex are *one motion*.

**The rotational family:**

- **Spinning** — 360°. A full turn.
- **Gyro** — 180°. A half turn.
- **Inspin** / **In-Spinning** / **In-Gyro** — the same rotations but the body turns *toward* the bag, not away from it. By default, spinning is a *backspin* (back passes the bag first). Inspin reverses that.

Spinning and Gyro can stack: Spinning Gyro is 540°, Gyro Gyro is 360°-and-back, Double Spinning is 720°. The language is compositional — combinations of half and full turns produce arbitrary degrees.

**Transformation rule:** `Spinning + [base] = compound with full-body rotation through the dex`. The base's uptime, dex shape, and catch are unchanged.

**Worked example: Spinning Whirl.** Whirl is `[set] > in dex > ss clipper`. Adding Spinning gives a full rotation through the dex moment: the set goes up, the body begins to rotate, the in-direction leggy dex happens during the rotation, the rotation completes, the catch lands on the same-side clipper. One continuous motion.

**Worked example: Mobius.** Now we can pay off the earlier promise. Mobius is a folk single-token name; its structural reading is "Spinning Torque." Torque is itself a structural reading — "Miraging Osis," an Osis base with a Mirage-style dex layered onto its rotational character. So Mobius decomposes as:

```
Mobius = Spinning Torque
       = Spinning (Miraging Osis)
       = Spinning + Miraging + Osis base
```

Three layers. The folk name "Mobius" compresses a three-layer composition into a single token. The compositional reading earns its visibility only now, after spinning and the rotational base concept have been introduced. That's the *earned compositional revelation* the primer's framing demands.

**ADD contribution:** +1 ADD for Spinning; +1 for Gyro. Stacked combinations sum.

**Relatives:** Gyro (the half-rotation sibling); Inspin (the direction-flipped sibling); the entire whirl-rotational topology (tricks that descend from rotational dexes share a structural neighborhood).

## §6.7 The paradox operator — the hip-pivot between two dexes

**What it does:** Paradox transforms the midtime window with a *hip pivot* — but specifically a hip pivot that connects two dexes on the same set. The body changes which side it's facing, and the leg returns to the same foot it left from.

**Body mechanics:** This one's worth slowing down on, because paradox is structurally the most precise of the body modifiers.

The setup: the set foot launches the bag. The kicking leg performs a *first* dex. The body pivots at the hips while the bag is still in the air — the chest turns, the support hip rotates, the kicking leg crosses to the other side of the body. The leg then performs a *second* dex, returning to the same foot the set launched from. The set, dex 1, hip pivot, dex 2 all happen on a single set, with the leg ending up at its starting foot.

This is why paradox isn't a head dip or a body spin: it's a *transition between two dexes*. The pivot fills the moment between them. The bag is in the air the whole time; the body is doing the work of carrying two dexes on one set.

**Why paradox is a hip pivot:** the freestyle community originally named the trick for the *paradoxical* feeling of returning to the same set foot after what feels like a longer trip. The body has done two dexes — the bag has traced a longer path — but the catch foot is where the set foot started. The hip pivot is what makes the geometry work.

**Transformation rule:** `Paradox + [base] = compound with a hip pivot inserted between the base's dex and a second dex on the same set`. Most paradox compounds end with the same-foot, opposite-direction-from-start catch.

**Worked example: Paradox Mirage.** A Mirage is `[set] > in dex > op toe`. Adding Paradox: the set, the in dex, then the hip pivot, then a second dex (typically in the opposite direction from the first), then the catch — back on the original toe. The trick is `[set] > in dex > (paradox hip pivot) > out dex > ss toe`. The catch is now on the same toe the set launched from, but the body has rotated and the bag has been through two dexes.

**Worked example: Paradox Whirl.** A Whirl is `[set] > in dex > ss clipper`. Adding Paradox gives a similar structure: set, dex 1, hip pivot, dex 2, catch. The base trick's clipper destination is maintained; the paradox is the operator that wedges the second dex in.

**ADD contribution:** +1 ADD.

**Cultural note on the name:** The shorthand `Pdx` is sometimes used in place of "Paradox" — particularly in compound trick names where space matters (e.g., `Pdx Whirl`, `Pdx Mirage`). Both refer to the same operator.

**Relatives:** Xdex (a similar hip-pivot mechanic but specifically for toe tricks rather than clipper tricks — its narrower sibling). Symposium (the other +1 body modifier, but transforming the support leg rather than inserting a second dex).

## §6.8 The symposium operator — the no-plant body modifier

**What it does:** Symposium transforms the *support leg's behavior* during a window. The support leg jumps and lands on its own — a single-leg jump — while the other leg performs the trick's dex.

**Body mechanics:** Normally, the support leg holds the body's weight throughout a trick. In a symposium component, the support leg hops *and* the other (active) leg performs the dex *while in the air*. The whole body is briefly airborne; both feet leave the ground; only the symposium leg lands again afterward.

The mechanic isn't a head dip or a body rotation — it's the *interruption of contact with the ground*. The trick's other components happen as usual; the symposium adds the layer of "this all happens in a single-leg jump."

**Transformation rule:** `Symposium + [base] = compound where the active dex is performed during a single-leg jump`. The dex shape is preserved; the no-plant character is added.

**Worked example: Symposium Whirl.** A Whirl is `[set] > in dex > ss clipper`. Adding Symposium: the set goes up, the leggy in dex happens — but during the dex, the support leg jumps. The Whirl is performed in the air. The catch lands; the support leg lands at the same time or just before.

**Worked example: PS Whirl = Paradox Symposium Whirl.** Now we're stacking. Both Paradox and Symposium transform the trick, but in different ways. Paradox inserts a hip pivot between dexes; Symposium makes the dex happen in a single-leg jump. Together they produce one of the canonical clipper-side compounds in freestyle: set, dex 1, paradox hip pivot, dex 2 — all of which happens during a single-leg jump (the symposium). The shorthand `PS` is the community's compression.

**Symposium variants:** The community has refined the no-plant concept into three closely-related operators:

- **Symposium** — the active dex happens in a single-leg jump; the symposium leg lands on its own
- **Symple** — starts as a symposium, but midway through, the non-symposium foot returns to the ground (a softer landing)
- **Muted** — the active leg is held in the air for the *entirety* of a component without putting it on the ground

The three are pedagogically distinct, but they're often used interchangeably as the shorthand `symp` — which can mean symposium OR symple depending on context. The glossary records the mechanics separately; the community shorthand stays a community shorthand.

**ADD contribution:** +1 ADD.

**Relatives:** Symple (the variant where the non-symposium leg returns mid-component); Muted (the variant where the leg never touches the ground); Paradox (the other +1 body modifier — often stacked, as in PS).

## §6.9 Atomic, Nuclear, and the set-modifier family

Now we can introduce the heavier set modifiers — the ones that don't just compress the uptime, but add *body* character to it.

### Atomic

**What it does:** Atomic transforms the uptime by adding a **cross-body character** to the set. Where stepping is a leg relocation and pixie is a compressed set, atomic adds a structural element — the leg's relocation involves crossing the body's centerline.

**Body mechanics:** The set foot doesn't just relocate; it crosses to the other side of the body during the uptime. This adds the xdex (cross-body dex) character to the uptime, which contributes both to the trick's compositional density and to its ADD count.

**Worked example: Atom Smasher = Atomic Mirage.** A Mirage with an atomic uptime: the set crosses the body, then the hippy in dex happens, then the catch on the opposite toe. The base trick's downtime is preserved; the uptime has gained a body-region transformation.

### Nuclear

**What it does:** Nuclear is the **paradox + atomic combination**. Per Red's pt10 ruling, Nuclear's structural reading is "paradox + atomic" — a +2 set modifier that combines the hip-pivot of paradox with the cross-body character of atomic.

**Worked example: Matador = Nuclear Butterfly.** A Butterfly with a nuclear set treatment: paradox's hip pivot, atomic's cross-body character, applied to the butterfly base. The result is one of the canonical 5-ADD compounds on a butterfly base.

**Worked example: Barfry = Nuclear ss Butterfly.** Same nuclear set, but on a same-side butterfly. The `ss` flag distinguishes it; the trick name carries the variant.

### Quantum

**What it does:** Quantum is the **compressed form of atomic** — a tighter, more compact version of atomic that fills the uptime with less travel.

**Worked example: Quantum Mirage = Toe Blur.** Quantum (the compressed atomic) applied to a Mirage. The trick has a historical folk name (Toe Blur) and a modern compositional name (Quantum Mirage).

### Furious

**What it does:** Furious is a **+2 set modifier** — a heavier uptime treatment that contributes two degrees of difficulty.

**Worked example: Fury = Furious Paradox Mirage.** Per Red's pt6 ruling, Fury's structural reading is "furious + paradox + mirage." Two compositional layers on top of the Mirage base. ADD math: 2 (furious) + 1 (paradox) + 2 (mirage base) = 5.

This is the kind of compositional decoding the operator framing makes possible. A folk single-token name like "Fury" decomposes cleanly into three layers, each layer's contribution accountable. The trick's ADD value follows from the operator math.

## §6.10 Timing modifiers — alpine, crispy

These operators don't add new content to a trick. They *reshape when content happens*.

### Alpine

**What it does:** Alpine takes a base trick that has both uptime and downtime components, and **inserts a duck or dive between them**. The base's structure is preserved; the alpine operator adds a head motion at the trick's midtime junction.

**Conditional branching:** Whether the inserted motion is a *duck* or a *dive* depends on the base's downtime side. If the downtime is op-relative-to-the-set, the alpine inserts a *duck*. If ss, the alpine inserts a *dive*.

**Worked example: Alpine Ripwalk = Stepping Ducking Butterfly.** Ripwalk is Stepping Butterfly. Its downtime (butterfly) is op-relative-to-the-stepping-set. So Alpine Ripwalk inserts a duck — giving "Stepping Ducking Butterfly," which is also a recognizable freestyle name.

**Worked example: Alpine Stepping ss Butterfly = Stepping Diving Butterfly.** Same base trick but with the ss variant. Now the downtime is ss-relative; alpine inserts a dive.

**What alpine does NOT do:** The operator only inserts ducks and dives — not the other head-motion siblings (weaving, zulu). A reader who knows the head-motion 2×2 might expect alpine to handle all four; the operator's domain is restricted to the two "simpler" siblings by community convention.

### Crispy

**What it does:** Crispy takes a downtime component of a base trick and **re-times it to the uptime**. Same motion; different timing window.

**Worked example: Crispy DLO.** A DLO is a Downtime Miraging Legover. Crispy DLO moves the mirage from downtime to uptime — producing what the community calls a Blurry Legover. Same body motions; different beat assignments.

**Cultural note:** Crispy is uncommon in canonical IFPA discussion. PassBack glossary records the term but flags it as often more confusing than just using the technical name. The V5 glossary records the operator but doesn't elevate it to the same prominence as paradox or spinning.

## §6.11 What just happened

If you read this section linearly to here, you crossed an important pedagogical threshold. You can now:

- Hear a compound trick name as a composition of operators on a base trick.
- Map each operator to the window of the timing clock it transforms.
- Reason about what the resulting body motion will feel like, based on which operators are stacked.
- Decode folk single-token names (Ripwalk, Phoenix, Mobius, Matador, Fury) into their structural readings.
- Recognize the same operator across multiple bases (paradox on mirage; paradox on whirl; paradox on torque) and know it's the same body action transferring to different operands.

**This is what "I can read tricks" means.** Not memorizing names; recognizing structure. The dictionary's compound entries — Phoenix, Matador, Mobius, Montage, Mullet, Mind-Bender, Gauntlet — are now readable to you. You don't need to memorize each one; you can decode them.

## §6.12 The compositional revelations, earned

Now that the operators are in place, we can name the compositions that the language enables. Each is *earned* by the operators we've met:

| Folk name | Structural reading | Operators stacked |
|---|---|---|
| **Ripwalk** | Stepping Butterfly | Stepping + Butterfly base |
| **Dimwalk** | Pixie Butterfly | Pixie + Butterfly base |
| **Phoenix** | Pixie Ducking Butterfly | Pixie + Ducking + Butterfly base |
| **Atom Smasher** | Atomic Mirage | Atomic + Mirage base |
| **Matador** | Nuclear Butterfly | (Paradox + Atomic) + Butterfly base |
| **Barfry** | Nuclear ss Butterfly | (Paradox + Atomic) + ss-Butterfly variant |
| **Toe Blur (or Quantum Mirage)** | Quantum Mirage | Quantum + Mirage base |
| **Fury** | Furious Paradox Mirage | Furious + Paradox + Mirage base |
| **Blur** | Stepping Paradox Mirage | Stepping + Paradox + Mirage base |
| **Torque** | Miraging Osis | Miraging-style dex + Osis base |
| **Mobius** | Spinning Torque | Spinning + Torque (i.e. Spinning + Miraging + Osis) |
| **PS Whirl** | Paradox Symposium Whirl | Paradox + Symposium + Whirl base |
| **Montage** | (Spinning + Ducking + Paradox + Symposium) Whirl | Four body modifiers on a Whirl base |

Each row reads as a sentence in the freestyle language. The base trick is the operand. The operators are the verbs. The compound name — the folk single-token name — is the cultural shorthand for the whole composition. Both names are valid; the compositional reading is what makes the language teachable.

Note that Montage at the bottom of the list is a four-operator stack: spinning + ducking + paradox + symposium, all on a Whirl base. Each operator transforms a different window of the trick's clock. Each operator is a +1 ADD body modifier. The total is Whirl(3) + 4 modifiers = 7 ADD, which matches Montage's canonical ADD value. The ADD math falls out of the operator framing.

## §6.13 Where this section's job ends

The dictionary owns the canonical catalog of compound tricks. This section's job is to give you the operator language for *reading* what the dictionary contains. When the dictionary lists a compound called *Spinning Paradox Symposium Whirl*, you should be able to:

- Identify the base (Whirl)
- Recognize each prefix as a body modifier (Spinning, Paradox, Symposium)
- Map each modifier to its window of the timing clock
- Sum the ADD contributions (3 + 3 = 6 ADD)
- Anticipate what the compound will feel like, based on the body mechanics of each operator

If you can do those five steps, you've internalized the operator framework. The rest of the glossary builds on this: §7 covers how to *write* what you can now *read*; §8 shows how compound tricks combine into strings and routines; §9 introduces the topology layer that lets you see operator patterns across base families.

## §6.14 What this section does NOT do

This section does not enumerate every modifier that has ever existed in freestyle vocabulary. The community has named many operators (PS, FS, BS, alpha/beta/gamma, alpine, crispy, etc.), some of which overlap. The V5 architecture chooses to:

- Teach the canonical operators (stepping, pixie, atomic, quantum, nuclear, furious, paradox, symposium, spinning, gyro, ducking and the head-motion family, alpine, crispy)
- Surface folk variants in §11 (community / historical vocabulary)
- Defer ADD scoring questions to §10 / §11
- Leave operator-ordering debates (which operator comes first in a name) to curator discretion

The section's job is teaching the *framework*. The exhaustive vocabulary list lives in the reference sections.
