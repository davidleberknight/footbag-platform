## 4. Glossary entries that are prose-heavy but mechanically vague

Three areas in PassBack glossary are educationally strong but mechanically vague — symbolic grammar fills the precision gap:

### 4.1 "Dex styles" (hippy vs leggy)

PassBack: "hippy = motion from the hip; thigh circles the bag; leggy = motion from the knee; calf circles the bag."

Symbolic grammar (CORE_TRICK_SYMBOLIC_TABLE): explicit mechanical mapping per canonical trick:
- mirage = `[set]>hippy in dex>op toe` (hippy-in canonical)
- illusion = `[set]>hippy out dex>op toe` (hippy-out canonical)
- legover = `[set]>leggy pass>op toe` (leggy single-pass canonical)
- ATW = `[set]>leggy circle>op toe` (leggy full-circle canonical)

The symbolic layer turns "hippy" / "leggy" from style descriptors into structural primitives. v4 entry should pair PassBack's prose with the SG primitives.

### 4.2 "Spins" (gyro / spinning / inspin)

PassBack: explains the degree system (180° / 360° / combine for arbitrary) and direction (backspin default; inspin = chest first).

Symbolic grammar: `spinning-family` modifier group; cuts across whirl / osis / torque / butterfly / clipper / drifter / blender bases; +1 ADD flat per pt10 ruling.

v4 entry should combine PassBack's degree taxonomy with the modifier-table ADD rule + the SG-2 modifier-family mapping.

### 4.3 "Ducks" (ducking / diving / weaving / zulu)

PassBack: the 4-variant table is mechanically precise (head-direction × bag-direction).

Symbolic grammar: `ducking-family` + `diving-family` are separate SG-2 modifier groups, but the 4 variants share a single body-mechanic axis. v4 entry should adopt the 4-variant table directly with cross-links to SG groups.
