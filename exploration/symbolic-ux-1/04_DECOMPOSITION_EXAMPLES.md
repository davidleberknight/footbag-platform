# SUX-1 Task D — Decomposition Visualization Prototypes

Four worked decomposition examples showing how symbolic-grammar layers expose a trick's structure across IFPA canonical reading + symbolic topology + modifier stacking + movement archetype.

**Date:** 2026-05-12

---

## Decomposition rendering grammar

Each decomposition uses a consistent visual grammar:

```
TRICK NAME (asserted ADD)
│
├── IFPA CANONICAL READING
│     base + modifier(s) per Red rulings or modifier-table math
│
├── SYMBOLIC TOPOLOGY
│     observational mechanical shape
│
├── MODIFIER STACKING
│     which modifiers compose, with ADD contributions
│
└── MOVEMENT ARCHETYPE
      reusable execution shape this trick exemplifies
```

All four examples preserve the four-layer separation: canonical reading sits in its own column; symbolic reading is observational; never conflated.

## Decomposition rendering rules

1. **Four sections per trick.** Canonical / Topology / Modifier-stacking / Archetype. Order is fixed for cross-page consistency.
2. **ADD math always shown.** Every numeric contribution explicit; reader can audit.
3. **Multi-description equivalence surfaced.** Where a trick has multiple equivalent readings (mobius, torque), all readings are shown without selecting one as "true."
4. **No IFPA family mutation.** `trick_family` referenced for context only.
5. **Modifier-stub filtering.** Sibling lists exclude the 18 modifier-stub rows.
