## 5. Generator-defect technical plan

Responsible: `freestyle/scripts/build_observational_universe_content.py`,
which classifies from `freestyle/inputs/observational/promotion_candidates_*.csv`
and `CLASSIFIED_UNIVERSE.csv` (May/June vintage) and never reads the ruling
ledger (`exploration/ev-formula-identity-audit-2026-07-10/EV_FORMULA_IDENTITY_ROWS.csv`).

Fix (classification authority, not a count patch):
1. The generator joins each observational row to the ledger by normalized
   name; WHERE THE LEDGER HAS THE ROW, the ledger's ev_state / disposition /
   blocker_subtype is authoritative and overrides the CSV-derived state.
2. Rows absent from the ledger render only after a live-database check:
   suppress from the authoring ladder any name resolving to an active
   canonical or a live alias (these display, if at all, as
   already-represented, never as backlog).
3. Object-type guard: rows whose ledger disposition marks a set/operator or
   not-a-trick-object (bare Nuclear class, disposition F set labels) never
   enter the authoring step.
4. Precedence when sources disagree: ledger > live DB alias/canonical check >
   observational CSV; disagreements emit a generator warning line so drift is
   visible at regen time.
5. Regression guard: extend the generated-content freshness gate with a test
   asserting (a) no authoring-ladder row resolves to an active canonical or
   live alias, and (b) every authoring-ladder row that exists in the ledger
   carries ledger state authoring/B or authoring/C. A small fixture pins one
   moved row (for example Zulu Far Legover) as suppressed.
