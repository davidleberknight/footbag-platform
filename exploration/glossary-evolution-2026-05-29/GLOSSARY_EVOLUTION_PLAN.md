# GLOSSARY_EVOLUTION_PLAN.md — Layered Architecture & Phased Roadmap (2026-05-29)

**Planning/audit sprint. No glossary edits, no doctrine resolved, no tricks promoted, no parser
changes, no ontology hard-coded.** Deliverable is this plan. It operationalizes the philosophy capture
in `../frontier-canonicalization-initiative/GLOSSARY_EVOLUTION_DIRECTION.md` into a concrete audit +
architecture + roadmap, and folds in this session's newer governance lessons (frontier calibration,
FM/Stanford lineage, two-band families, media claim-scope, the illusioning/rev(0) resolution).

**What the glossary is becoming.** Not just a trick dictionary or movement primer, but a *layered
explanation system* that teaches **how freestyle movement language works** — how structures decompose,
how folk and structural names project the same movement, how advanced names compress, how equivalence is
resolved, where doctrine is still open, and how documentation differs from canonization. The unifying
idea: **folk names and structural decompositions are two projections of the same underlying structure.**

---

## PART A — Current glossary audit (findings only)

The glossary is already more mature than "a primer." It has tiered progressive disclosure
(Beginner / Intermediate / Advanced Reference badges) and three load-bearing **seeds** the new layers
should extend rather than replace.

### A1. What works (keep)
- **Beginner spine is solid:** "How to read this glossary" intro card · "Reading the dictionary" (trick
  row / six browse views / five kinds of object) · "Movement Basics" + the twelve core atoms · Surfaces ·
  Dexterities (motion style / direction / window / quality / ducks). Concrete, example-led, well-tiered.
- **Structural seed exists:** the "compositional premise — every trick as formula (Job, 1995)" + Jobs
  notation + generative-insight sections already teach decomposition.
- **Equivalence seed exists:** "Vocabulary relationships — four ways trick names can relate" + the
  walking-family worked example.
- **Governance seed exists:** "Tracking is not canonization" already states documentation ≠ canonization,
  observational readings exist, and a name becomes canonical only by promotion-after-ruling. This is
  Layer 5 in miniature and is the strongest existing hook.
- **Family doctrine exists:** "What makes a family?" / parent families / descendant lineages / "lineage
  or ecosystem? a test you can apply."

### A2. Stale or incomplete
- **Family doctrine predates the terminal-identity / minor-derived distinction** (stabilization sprint).
  It does not yet separate *terminal-identity families* from *minor & derived neighborhoods*, nor frame
  entry-side primitives/operators (ATW) as non-families.
- **No frontier-governance layer.** The calibration lessons (observational counts overstate; frequency ≠
  authority; lineage collapse; independent corroboration) are absent. "Tracking is not canonization" is
  the only governance content and it is a single short block.
- **No media claim-scope content.** Nothing explains tutorial vs demo vs record, or that media teaches
  movement without settling structure.
- **No high-ADD / exotic-name interpretive section.** The glossary teaches decomposition as if it always
  resolves cleanly; it does not explain why advanced names become opaque or doctrine-dependent.

### A3. Where it overstates certainty
- Compositional/decomposition sections present readings as if uniformly settled; doctrine-sensitive
  decompositions (blurry, DOD, atomic-Q3) are not *marked as open*.
- `trick_family` is presented as a single clean axis; multi-axis membership and disputed family
  assignment aren't acknowledged (skill §C multi-axis caution).

### A4. Where it underexplains structural interpretation
- Folk-vs-structural naming is implied by "vocabulary relationships" but not taught as **two projections
  of one structure** — the central unifying idea.
- Equivalence is described by *category* (four ways names relate) but not by *mechanism* (decomposed-
  structure matching, not name-string matching; why one canonical surface is chosen).
- Compression ("when the long reading IS the trick") is hinted at but not a first-class concept.

### A5. Layer confusions to fix
| Confusion | Current state | Fix layer |
|---|---|---|
| canonical vs observational | only "tracking is not canonization" | L5 frontier governance |
| trick vs operator | "other foundational atoms" mixes them; ATW framing | L2 (terminal identity) + L5 |
| family vs neighborhood | no minor/derived band | L2 (two-band families) |
| tutorial vs evidence | absent | L6 media claim-scope |
| folk name vs structural decomposition | implicit in "vocabulary relationships" | L3 (two projections) |

### A6. High-ADD complexity: currently unsupported
No content explains why high-ADD/exotic-name tricks expose the limits of flat symbolic language, why
compression increases ambiguity, or why curation gets harder (and more doctrine-dependent) as ADD rises.

## PART B — Layered architecture (6 layers, mapped to what exists)

Progressive disclosure: surface stays beginner-friendly; governance depth lives deeper. Each layer notes
**EXISTS / EXTEND / NEW** against the current glossary.

| Layer | Content | Status vs current glossary |
|---|---|---|
| **L1 — Beginner movement language** | stalls, dexes, sets, uptime/downtime, basic ADD, core tricks, readable examples | **EXISTS** (Movement Basics, Surfaces, Dexterities, core atoms) — keep |
| **L2 — Structural decomposition** | compound reading, operators/modifiers/primitives, family-generator logic, **terminal identity**, compositional formula; examples ripwalk/barrage/blender/mobius | **EXTEND** (compositional premise + families exist; add terminal-identity framing + two-band families) |
| **L3 — Names & equivalence** | **folk vs structural as two projections**, aliases, canonical surfaces, equivalence chains (decomposed-structure matching), why a name collapses vs stays distinct; gauntlet / alpine-big-apple / rev-up↔reverse-whirl | **EXTEND** (vocabulary-relationships seed → add the projection framing + equivalence mechanism) |
| **L4 — Compression & high-ADD interpretation** | why advanced names densify, hidden structure, symbolic shorthand, when multiple readings are plausible, why high-ADD curation is harder | **NEW** (Part C) |
| **L5 — Frontier governance** | observational vs canonical, parser-derived vs culturally attested, lineage collapse, independent corroboration, single-source caution, frequency ≠ authority, doctrine-gated regions | **EXTEND** ("tracking is not canonization" seed → a full arc) |
| **L6 — Media claim-scope** | tutorial vs demo vs record, media as teaching layer not ontology, teach-without-resolving, future teaches/components-covered edge | **NEW** |

**Reading-depth contract:** a beginner can stop after L1–L2 and have a complete, useful mental model. L3
is for the curious; L4–L6 are advanced/expert and explicitly badged. No L4–L6 concept is a prerequisite
for using the dictionary.

## PART C — High-ADD / exotic-name interpretive section (design)

**Section working title:** "When movement language gets hard — high-ADD & exotic names." Advanced-badged,
deep in the reading order. Teaches that **advanced freestyle language is partly interpretive and
governance-dependent** — symbolic interpretation itself becomes part of expertise.

**Concepts:** why folk names become opaque · why structural names become long · why compressed notation
can hide assumptions · why equivalence is sometimes clear and sometimes disputed · why doctrine matters
more at high ADD · why some tricks stay safely queued instead of promoted · how videos are evidence, not
doctrine.

**Case studies** — each tagged with publish-safety (drives Part D and the roadmap):

| Case | Teaches | Publish status |
|---|---|---|
| **gauntlet ↔ stepping-ducking-paradox-torque** | folk ↔ structural projection; clean decomposition collapse | **SAFE NOW** (verified clean) |
| **alpine-big-apple ↔ gyro-ducking-symposium-torque** | structure-not-string equivalence; alias-wiring | **SAFE NOW** (verified + wired this session) |
| **rev-up ↔ reverse-whirl; illusion ↔ rev(0) mirage** | direction as structure; the rev(0) operator | **SAFE NOW** (rev(0)/illusioning resolved this session) |
| **flailing / illusioning (operator-vs-trick boundary)** | gerund-of-base operator question; one resolved, one open | **PARTIAL** — illusioning resolved (publishable as rev(0) miraging); flailing **HOLD** (open) |
| **gangsta-party vs spinning-ducking-paradox-blender** | same-structure / multi-name; verify-before-publish | **HOLD until verified** (do not assert equivalence as settled) |
| **blurry-transitivity cluster** | when a decomposition is *disputed* (is blurry ≡ stepping+paradox?) | **HOLD / mark open** until Red Wave 2 |
| **DOD cluster** | policy-dependent ADD interpretation | **HOLD / mark open** until Red Wave 2 |
| **atomic / Q3 cluster** | hidden-carry / X-Dex on rotational bases | **HOLD / mark open** until Red Wave 2 |
| **superdeeduperfly** | a trick *held* because doctrine (DOD) is unresolved | **HOLD / illustrate gating** (internal until ruling) |

**Doctrine-honesty rule for this section:** the HOLD cases render as **open questions** ("the community
hasn't settled whether…"), never as resolved equivalences. They are the section's *point* — they teach
that uncertainty is normal scholarship, not system weakness.

## PART D — Public wording strategy

**1. Suggested section titles** (beginner-plain on top, governance terms only deep):
- L1: "Movement basics" · "Reading a trick" (existing)
- L2: "How tricks are built" · "Families & terminal identity"
- L3: "Names: folk and structural" · "When two names mean the same move"
- L4: "When movement language gets hard"
- L5: "How we decide what's canonical" (advanced) — subhead "Documented isn't the same as settled"
- L6: "What a video can and can't tell you"

**2. Suggested sidebar grouping** (three bands, mirroring the tier badges):
- *Start here* (L1–L2 beginner) · *Going deeper* (L2–L3 intermediate) · *Interpretation & governance*
  (L4–L6 advanced). The third band is collapsible and clearly labeled "advanced / reference."

**3. Beginner-facing intro language** (no jargon):
> "Most of this glossary is about *doing* tricks. The later sections are about *reading* them — how
> trick names are built, why the same move sometimes has two names, and how we decide which name is the
> official one. You don't need any of that to start; come back when you're curious."

**4. Advanced-facing governance language** (honest, not bureaucratic):
> "Some advanced tricks have more than one reasonable structural reading, and the community hasn't
> settled all of them. We mark those openly rather than pick a winner prematurely. A name in our records
> is evidence it exists in the culture — not a claim that its structure is final."

**5. Examples safe to publish now:** gauntlet ↔ stepping-ducking-paradox-torque; alpine-big-apple ↔
gyro-ducking-symposium-torque; rev-up ↔ reverse-whirl; illusion ↔ rev(0) mirage / illusioning; the
*principles* of frontier governance (frequency ≠ authority; documented ≠ canonical; independent
corroboration) — stated as principles, without the internal calibration numbers.

**6. Examples to keep internal until Red Wave 2:** blurry-transitivity, DOD-policy, atomic/Q3 as
*resolved* readings; superdeeduperfly's final disposition; gangsta-party structural equivalence (until
verified); any specific frontier counts / lineage-weighting internals. These may be *referenced as open
questions* but never published as settled.

## Final — the five answers

1. **What the glossary is becoming:** a layered explanation system — beginner primer at the surface,
   structural-interpretation + equivalence + governance framework underneath — that teaches *how
   freestyle movement language works*, not just *what tricks exist*.
2. **Change now vs later:** *Now* — wording fixes (Phase 2), the L2/L3 structural+equivalence extension
   on verified examples (Phase 3), the governance *principles* arc (Phase 4), and media claim-scope
   (Phase 6). *Later* — the high-ADD interpretive section's doctrine-sensitive case studies (Phase 5) and
   any *resolved* forms of the open clusters.
3. **What must wait for Red Wave 2:** the settled readings of blurry-transitivity, DOD-policy, atomic/Q3;
   superdeeduperfly's disposition; the gangsta-party structural equivalence (pending verification). These
   appear *only as open questions* until ruled.
4. **Teaching uncertainty without confusing users:** keep beginners on L1–L2 (zero governance jargon);
   badge L4–L6 advanced and collapse them; frame open regions as *careful scholarship* ("not yet
   settled, on purpose"), never as system weakness; always pair an open question with the reason it's
   open. The existing "Tracking is not canonization" block is the tonal template.
5. **The glossary as bridge:** it becomes the single legible surface where the **dictionary** (canonical
   names), the **parser/decomposition** (structural readings), the **media** (claim-scoped evidence), and
   the **governance model** (observational/canonical, corroboration, doctrine-gating) meet — linking the
   four layers for the reader without collapsing them (the four-layer ontology holds).
