# Curator Packet — Pogo ADD Doctrine

## Why this packet exists

The footbag.org historical corpus contains a substantial pogo family spanning the
3–6 ADD tiers. These tricks are currently tracked in the observational universe but
remain unpromoted because of a single unresolved question:

**Does pogo contribute ADD?**

The answer determines the ADD values for an entire family of tricks and whether their
published footbag.org counts should be preserved or normalized.

This packet does not ask whether pogo exists as a movement. It clearly does. The
question is only whether pogo is a count-bearing element.

---

## Current tension

### Position A — Red doctrine

Current Red guidance:

> Pogo is a set and does not add ADD.

Under this reading:

```text
pogo = +0
```

Pogo is treated similarly to other non-count-bearing entry conditions.

### Position B — Historical footbag.org treatment

Footbag.org consistently lists pogo compounds one tier above their pogo-less base,
which is only consistent with pogo contributing +1.

## What is settled

* Pogo is a recognizable movement pattern.
* Pogo compounds are documented in the historical record.
* The affected tricks are already tracked in the observational universe.
* No decomposition dispute currently exists beyond the pogo contribution itself.

---

## What is open

### Ruling 1 — Should pogo contribute ADD?

#### Option A — Pogo = +0

Interpretation:

```text
pogo = set only
```

Consequences:

* Existing Red doctrine remains unchanged.
* All pogo compounds derive directly from their underlying base tricks.
* Historical footbag.org ADD values become legacy counts rather than canonical counts;
  the entire family drops one tier from its published value.

Example:

```text
Pogo Butterfly = Butterfly ADD (3)
```

rather than the published 4.

#### Option B — Pogo = +1

Interpretation:

```text
pogo = count-bearing entry condition
```

Consequences:

* Historical footbag.org counts are preserved (the corpus arithmetic above already
  assumes this throughout).
* Entire pogo family promotes using a straightforward compositional rule
  (base + pogo(+1), with paradox(+1) where present).
* Pogo becomes a first-class +1 modifier alongside other count-bearing entry systems.

## Recommendation

The historical evidence clearly establishes that the pogo family exists, and the corpus
arithmetic uniformly places every pogo compound at base + 1.

The remaining question is whether that movement represents:

1. A descriptive set condition (+0), in which case footbag.org's published counts are
   all legacy and the family normalizes one tier down, or
2. A count-bearing compositional modifier (+1), in which case the published counts are
   canonical and the family promotes by a single compositional rule.

The ruling should be made at the operator level and then applied consistently across the
entire family rather than adjudicating individual pogo compounds one at a time.

The one point of friction to resolve explicitly: Position B reverses the current Red
guidance ("pogo is a set and does not add ADD"). If Red affirms +0, the corpus counts
are overridden as legacy; if Red affirms +1, the prior guidance is superseded. Either
outcome is consistent and promotable; what blocks promotion today is only the
unresolved choice between them.
