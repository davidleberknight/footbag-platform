## The doctrine is settled and in the live data

Red finalized the Atomic / X-Dex ADD doctrine and the ADD-value migration has shipped (the old
`atomic +2-on-rotational-bases` model is retired). The current `atomic` modifier registry states it:

> Atomic = +1 ADD on every base; it has no rotational class (the +2-rotational reading is retired).
> Any count beyond +1 comes only from a separate conditional X-Dex (+1), which fires solely on a far
> move after a non-paradox dex on an eligible receiver (mirage, illusion, whirl, torque, drifter),
> and is recorded only where the notation carries `[XDEX]`.

The canonical ADDs apply it consistently: `atom_smasher` (atomic **far** mirage) = 4 with `[XDEX]`;
near/default forms do not fire (`atomic_whirl` = 4, `atomic_torque` = 5); non-receivers do not fire
(`atomic_swirl` = 4, where the retired +2-rotational model wrongly gave 5). So **whether X-Dex fires
on an atomic far receiver is not an open question** — it is ruled: yes, on a far mirage / illusion /
whirl / torque / drifter, recorded via `[XDEX]`.
