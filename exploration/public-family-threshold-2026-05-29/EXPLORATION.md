# Public "By family" surface — family-base qualification exploration

Read-only UX/ontology exploration. **No canonical deletion, no code change, no edit to
`freestylePublicFamilies.ts`.** Explores how to refine which of the curated 23 public families are
presented as first-class "By family" roots. Reversible direction-setting for curator decision.

## Principle: count is a demotion *signal*, not a doctrine rule

A low trick count does NOT automatically disqualify a public family. Count is one signal that *triggers
an assessment*. The real test is:

> **Does this grouping behave like a family base** — a terminal structural topology that other tricks
> *inherit* and stack modifiers onto — rather than a derivative micro-cluster or a component primitive?

Assess each candidate on: (a) trick count, (b) structural uniqueness, (c) whether it **generates
modifier-stacked descendants**, (d) whether it is **derivative of another family**, (e) **terminal /
downside structural identity** — does the movement resolve into a recognizable *landing* signature?,
(f) **persistence under modifier stacking** — do descendants still "feel like" the base structure when
modifiers stack onto the *entry*?, (g) whether tricks **inherit it as terminal topology** vs merely
**use it as an entry-side component**, (h) pedagogical value.

The heart of the test is (e)+(f): **a true family base is a terminal/downside structural identity the
trick lands into, and that descendants preserve even as modifiers stack on the entry.** Whirl, osis,
butterfly, mirage, drifter, torque, barrage all behave this way — the trick "lands" into a recognizable
structural signature, and `paradox-whirl` / `spinning-osis` still feel like whirl / osis. A grouping
that lacks an inherited terminal signature — one that is *used on the entry* rather than *landed into* —
is an operator / component / primitive, not a family. This is precisely why ATW is not a family root
(Cat 4).

## Four categories

1. **Productive high-count topology roots** — clear public family bases. (whirl, mirage, butterfly,
   legover, osis, illusion, pickup, drifter, double-over-down, barfly, flail, eclipse, barrage, dyno,
   paradon, butterfly-swirl, dada-curve.)
2. **Sparse but structurally distinctive families** — low count but valid public roots: unique
   structure that generates modifier-stacked descendants and is pedagogically meaningful. **Keep public.**
3. **Sparse derivative / micro-clusters** — low count AND derivative of a parent / a motif. **Demote**
   to derivative structures, motifs, or sub-families (still reachable, not equal-status).
4. **Primitives / components / movement-system roots** — foundational and important, but tricks *use*
   them rather than *inherit* them as terminal topology. **Reclassify out of the family lens** into
   components / dexterity primitives / movement systems / educational foundations. (Not a family at all;
   not a deletion.)

## around-the-world (ATW) — reclassify, do not demote-as-family

ATW is historically foundational, pedagogically central, and a core movement/component/operator pattern
— but it does **not** behave like a productive topology family. Its only "descendants" are count-variants
(double-ATW, triple-ATW) and a kick; there is no modifier-stacking ATW family the way mirage/whirl/osis
spawn paradox-/spinning-/symposium- compounds. Most tricks **use** ATW as a dexterity component, they do
not **inherit** "ATW" as their terminal structural topology.

So ATW belongs conceptually with **components / dexterity primitives / movement-system archetypes /
educational foundations**, not first-class public "By family" roots. This is a reclassification to a
different layer, NOT a deletion — ATW remains extremely important and prominent, just not under the
family lens. (Its `atw` / `around-the-world` label split is a separate data-reconciliation item; resolve
it wherever ATW lands.)

## What the surface becomes

- **Family roots band (Cat 1 + Cat 2):** the 17 productive roots + flurry (and likely superfly,
  curator call) — clean, each a genuine inheritable topology anchor.
- **Minor / derived families band (Cat 3):** reverse-drifter, down-double-down, infinity — visually
  separated, reachable, framed as derivative structures / sub-families under their parent.
- **Reclassified out of the family lens (Cat 4):** around-the-world → components / movement-system /
  foundations surface.

This removes the foundational-root vs sparse-derivative blur AND the family-vs-primitive blur, sharpening
the pedagogical clarity of the family lens.

## Recommendation

1. Treat count as a **demotion signal that triggers the behaves-like-a-family-base assessment**, not an
   automatic rule. `flurry` (3) stays public on that basis; `dada-curve` (4) is a clear root.
2. **Demote Cat 3** (reverse-drifter, down-double-down, infinity) to minor/derived families under their
   parents; `superfly` is a curator borderline call.
3. **Reclassify ATW (Cat 4)** out of the family lens to the component / movement-system / foundations
   layer; reconcile its `atw`/`around-the-world` label there.
4. Implementation (future, curator-approved): a `tier`/`role` field on `freestylePublicFamilies.ts`
   (root / minor-derived / reclassified-primitive) + a two-band rendering split (rendering is Dave-track).

## Topology roots vs operators / components / primitives (the core discriminator)

The sharpest distinction this exploration surfaces: **a family / topology root is a terminal structural
identity that tricks land into and preserve under modifier stacking; an operator / component / primitive
is an entry-side movement that tricks use without inheriting a terminal signature.** Descendant tricks of
a real family still "feel like" the base (paradox-whirl is still a whirl; spinning-osis is still an osis)
because the recognizable landing topology persists while modifiers stack on the entry. ATW fails this:
it has no inherited terminal signature, so it is a dexterity/operator primitive, not a topology root.

**Forward thread — glossary philosophy (not actioned here):** this terminal-identity test should inform
the glossary §5 family doctrine and family-tree. Public families should be framed by *terminal-identity
persistence under stacking*, and primitives/components/operators (ATW, dexterity primitives, set/uptime
operators) should be presented on their own footing — explicitly NOT as family roots. Capture in the
glossary work when taken up; it unifies the family lens, the operator/modifier lens, and the movement-
system lens around one principle.
