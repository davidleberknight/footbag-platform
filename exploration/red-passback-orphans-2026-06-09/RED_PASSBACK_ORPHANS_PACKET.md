## The decision per label (please confirm this is the right frame)

For each PassBack name we plan to file it as exactly one of:

(a) **new canonical** — a real trick we do not have yet (we canonicalize it; the PassBack name is the name)
(b) **alias of <existing trick>** — same trick, different name (the PassBack name becomes a searchable alias / descriptive sub-name)
(c) **historical variant** — a genuinely used older name we document but do not canonicalize
(d) **not a trick** — record category, notation fragment, or titling noise

Two questions on the frame itself:

1. Is that the right four-way?
2. When PassBack's technical decomposition matches a trick we already have, should we treat the
   PassBack name as an **alias by default** rather than a new canonical, i.e. is the decomposition
   authoritative enough to map on?

## Group A — recurring operators (one ruling each settles a whole cluster)

Most orphans are built from operators you have already settled, so these are mostly confirmations:

**A1 — Miraging / Illusioning.** Your June note settled miraging as a dexterity operator (X-Dex on
the second dex), and illusioning as rev(0) miraging is already in. About 17 orphans lead with one
of these (for example **Anonymous** → `Spinning far Miraging Symp. Miraging Refraction`). Confirm
we apply that ruling straight through this cluster?

**A2 — Barraging.** 8 orphans lead with "Barraging" (for example **Archnemesis** → `Barraging near
Double Down`, dex 4). We currently treat barraging as a settled two-dex operator. Same operator
here, map mechanically?

**A3 — Undefined terminating tokens.** A few decompositions bottom out in elements we have no base
for: **Refraction** (in Anonymous), **Flux** (Big Orange → `Symp. Flux`), **Magellan** (Alpine
Magellan, BS Magellan). Are these real terminating elements you can define, or folk labels for
something we already have?

## Group B — competing decompositions (43 labels)

43 orphans carry two technical readings. For example **Atom Bomb** → `Flailing far Symp. Mirage`
OR `Full Symp. Atomsmasher` (and we already have atom-smasher canonical). Per "one true name":
when two readings describe the same trick, do we pick the one matching an existing canonical and
treat the other as the sub-name, or would you rather rule per case? A general rule resolves all 43.

## Group C — notation / concept leakage (confirm not tricks)

A handful of "labels" read like header rows or dex-notation rather than tricks: `>`, `Dex`,
`Dexterity`, `Contact`, `In(-Out) Dex`, `Out(-In) Dex`. The same goes for bare-operator fragments
that never reach a terminating element: `Gyro`, `Spinning`, `Double Spinning`. We read all of
these as **not tricks**. Confirm we drop them from the trick surface?
