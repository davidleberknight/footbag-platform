# Glossary evolution — forward direction (NOT for immediate implementation)

A design-philosophy capture, not a work item. Records the direction the glossary should eventually
evolve toward, so it can shape future glossary architecture + explanatory philosophy. **Nothing here
is built now; no glossary edits.** When glossary work is next taken up, read this alongside the
frontier doctrines (`CHARTER.md`, `PHASE1`/`PHASE2`) and the family-base doctrine in the
`footbag-freestyle-dictionary` skill (§C).

## The conceptual shift

The project is no longer merely a **trick dictionary** or a **movement-language teaching system**.
It is also becoming:
- a **structural interpretation framework** (how a movement decomposes),
- an **equivalence-resolution system** (when two names denote the same structure),
- and a **canonicalization / governance model** (how the community's vocabulary is responsibly governed).

The glossary is the surface where these eventually become legible to a reader. Its center of gravity
shifts from *"what tricks exist?"* toward *"how movement structures are interpreted, compressed,
related, and canonically governed."*

## The critical insight to teach

**Folk names and structural decompositions are not separate worlds — they are different projections of
the same underlying movement structure.** A folk name is one projection (cultural/identity); the
operator-chain decomposition is another (structural). The same trick can carry both. This is the idea
that unifies the dictionary, the operator/modifier system, the alias layer, and the frontier-governance
work.

Worked examples the glossary should eventually use to make this concrete:
- `gauntlet` ↔ `stepping-ducking-paradox-torque` (folk projection ↔ structural projection; same trick).
- `alpine-big-apple` ↔ `gyro-ducking-symposium-torque` (same; verified + alias-wired this session).
- `blurry-transitivity` as **unresolved doctrine territory** (is `blurry` ≡ `stepping + paradox`?).
- **DOD-family policy dependence** (the double-over-down ADD-interpretation question).
- **atomic / Q3 doctrine clusters** (atomic-on-rotational +2 / X-Dex carry).

## What a future glossary should explain

A mature glossary should make these first-class, teachable concepts:
- **Structural decomposition** — a trick as an operator chain over a base (the modifier-set + base).
- **Symbolic compression** — when a long structural reading IS the trick (the folk name compresses it).
- **Folk-vs-structural naming** — the two projections; why both exist and how they relate.
- **Equivalence chains** — how names resolve onto the same structure (decomposed-structure matching, not
  name-string matching).
- **Parser / frontier uncertainty** — that some readings are computed-confident and others are not.
- **Doctrine-sensitive regions** — areas where canonical ADD/decomposition is still being ruled (blurry,
  DOD, atomic-Q3); the glossary should *mark* these as open, not pretend they're settled.
- **Why some tricks collapse structurally while others remain unresolved** — the difference between a
  clean decomposition (gauntlet) and a doctrine-blocked / FM-only / above-7 frontier candidate.

## Governing principles this evolution must keep

- **Layer separation holds.** Canonical names / symbolic decomposition / glossary pedagogy / embodied
  analogy stay distinct (the four-layer ontology). The glossary *links* them; it does not collapse them.
- **Mark uncertainty, don't hide it.** Doctrine-sensitive regions render as open questions (the
  observational-layer posture), never as false certainty.
- **Reversible, curator-paced.** Glossary content is curator-authored; this is explanatory philosophy,
  not a schema or parser change.
- **Terminal-identity framing** (skill §C): families are terminal-identity structures; primitives /
  operators are entry-side. The glossary's family doctrine (§5) should eventually adopt this framing and
  present primitives/operators (ATW, dexterity primitives, set/uptime operators) on their own footing.
