# Decision on the Next Research Step

Status: exploratory research under `exploration/`. Not production data, not doctrine, not a change to the live dictionary. It chooses one next research step from the four offered and justifies it from the evidence of this interlude. It commits to nothing beyond the recommendation and starts no work.

## The four options

1. **Enrich the grammar's handoff state** (add the state vocabulary and transition function the gap report identifies).
2. **Validate the state model against existing two-dex notation** (test the proposed four-component handoff state against the notation the corpus already records).
3. **Consult expert or video evidence.**
4. **Proceed with a bounded atlas using unresolved-state markings** (map the two-dex space now, marking every undecided pairing).

## The recommendation

**Option 2: validate the proposed handoff-state model against existing two-dex notation, before anything else.**

## Why, from the evidence

The interlude produced a **proposal**, not a finding: a four-component handoff state whose confidence is high that the components are required and low that they are recoverable from current notation. A proposal in that posture has exactly one correct next move, which is to test it against the evidence already in hand before building anything on it. Three facts make validation the disciplined choice.

- **The model is untested.** The closure and transport tests showed the notation is an event language that does not carry the state, so the proposed state is inferred from the definition of a dexterity, not read from the tokens. Whether that inferred state is sufficient to explain the composability the notated two-dex movements actually exhibit, and whether the reference frame in fact turns in those movements, are answerable by reading the notation the corpus already holds. That reading is the missing evidence.

- **Every other option presupposes the model.** Enriching the grammar (option 1) would add tokens for a state that has not been shown to be the right state; it is the repair the task explicitly forbids for now, and it is premature before validation. A bounded atlas (option 4) would freeze a map whose "unresolved" markings are only as trustworthy as the unvalidated state model that generated them; if the model is wrong, the markings are wrong. Both build on the proposal instead of testing it.

- **Validation stays inside the allowed lane.** It uses only the grammar and its notation: reading the structural tokens of movements that already carry two dexterities, checking whether the four-component state is recoverable from them, whether the frame transports, and whether the composability classes hold. It needs no video, no expert ruling, and no physical-performance claim, so it respects every boundary of this interlude. Option 3 (expert or video) is the right escalation **only if** validation against notation proves inconclusive, because expert and video evidence answer physical and community-usage questions that sit outside the structural layer this interlude is confined to.

## The sequence this implies

Validate first (option 2). If validation confirms the four-component model and shows whether the frame turns, then enriching the grammar (option 1) becomes warranted and well-specified, and a bounded atlas (option 4) can be built on trustworthy markings. If validation is inconclusive because the notation itself does not carry enough state to test the model, that is the precise trigger for consulting evidence outside the notation (option 3). The bounded atlas (option 4) is the correct fallback deliverable at any point, but it should carry the validation's verdict, not precede it.

## What this decision is not

It is not a commitment to begin validation now; this interlude starts no follow-on work. It is not a repair of the grammar, not a Volume III, and not a restoration of the 64-point count. It is a single, evidence-grounded choice of what the project should do next, and its whole content is: **test the handoff-state model against the notation we already have before building on it or around it.**
