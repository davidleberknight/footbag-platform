# Recoverable Structure Audit - the 705 Folk / Unresolved

Read-only. Re-audits the 705 folk/unresolved observational structures by **recoverability**, not by the stale Phase E bucket. Method: tokenize each structure's NAME against the *currently ratified* family / modifier / set / operator / composition grammar (the parser at Phase E, 2026-05-28, did not have it). The question is not "is a decomposition stored" (none of the 705 have one) but "can a structure be recovered today from evidence already in the project."

## Headline answer
**283 of the 705 (40%) are near-term recoverable** - far more than the "folk/unresolved" label implies:
- **A - grammar-derivable NOW: 172.** The name tokenizes entirely to ratified operators + a known base; the structure is recoverable today with zero new rulings.
- **C - blocked by one un-ratified operator: 111.** Otherwise fully tokenized; a single operator ruling clears each cluster.
- **D - genuine doctrine question: 16.**
- **E - truly unresolved (opaque name, no recoverable structure): 406.** Need source evidence (FootbagMoves / Holden / PassBack / tutorials), a real authoring task.

The Phase E classification put all 705 in "unresolved." The sprint's notation/grammar/operator/composition work makes 172 immediately recoverable and 111 one-ruling-away.

## Why the prior audit understated this
The earlier readiness audit treated these as "decomposition authoring (691)" because no decomposition is *stored*. But a decomposition does not need to be stored - it can be **recovered from the name** when the name is compositional and its tokens are now ratified. `failureClass` proves it: 243 were "unknown-modifier-token" and 9 "unresolved-directional-syntax" - i.e. the Phase E parser rejected a modifier/directional it didn't know, and the sprint has since ratified many of them (pixie, stepping, fairy, sailing, railing, spyro, reverse, double/triple, inward).
