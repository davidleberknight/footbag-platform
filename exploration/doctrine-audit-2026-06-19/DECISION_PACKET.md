## Section A. Doctrine questions

### A1. DOD / DDD  (~25 tricks)

> **Superseded 2026-07-02** by `exploration/dod-ddd-evidence-pass-2026-07-02/RED_PACKET_A1_REVISED.md`.
> Distinctness is no longer the live question (evidence strongly favors distinct, all three
> original ruling options below are stale). The unresolved issue is base identification when
> dex-inserting operators (atomic, quantum, symposium, shooting, nuclear) have altered the JOB.

**Question.** Is DOD (Double-Over-Down) the same structure as DDD (Down-Double-Down), or two
distinct canonical bases? And is "down" a base or a naming layer?

**Evidence.** Both read **4 ADD** as bare bases yet are kept as separate rows. DOD anchors a
large settled compound family (fusion = atomic-DOD = 5, plasma = quantum-DOD = 5, the pixie/
gyro/fairy/stepping/symposium/tapping-double-over-down set). DDD is sparsely populated
(scorpions-tail = spinning-DDD = 5). The compression chain superfly -> superduperfly ->
superdeeduperfly encodes the same spinning/ducking stack the structural names already carry,
i.e. the down-naming looks like a folk layer over the operator stack.

**Ruling options.** (A) DOD == DDD: pick one canonical base, the other becomes an alias.
(B) Distinct: state the over-down vs double-down structural rule so the band decomposes to the
right base. (C) "Down" is a naming layer: decompose each compound to its operators, no DOD/DDD
base.

**Affected (live ~25).** symposium/diving/nuclear/quantum/sailing/shooting-down-double-down +
atomic/pixie/spinning/stepping/tapping far/near double-down chains.

### A2. Weaving  (~30 tricks)
**Question.** Is weaving a productive operator (+N), a movement family (+0), or a naming layer?

**Evidence.** Registered placeholder +1, but **no notation authored for any weaving compound and
no source corpus gives its ADD.** It spans both stall bases (clipper, osis) and dex bases
(mirage, illusion, legover, pickup), which is the spread of an operator rather than one trick.

**Ruling options.** (A) Operator with a stated +N -> whole family promotes compositionally.
(B) Movement family +0 -> derive from base. (C) Naming layer -> define the equivalent stack.

**Affected (live ~30).** weaving-{butterfly, clipper, legover, osis, pickup, mirage, illusion,
drifter, eclipse, guay, symposium-mirage, xbd-rake, magellan(aliased), pigbeater(built)} + ~15
stacked fairy/pixie/gyro/quantum/stepping/tapping-weaving variants.

### A3. Pogo  (~24 tricks)
**Question.** Does pogo contribute ADD, +0 (current Red position) or +1 (corpus)?

**Evidence.** Red's standing position is +0. Footbag.org lists **all 13 pogo compounds exactly
+1 above their pogo-less base, zero counter-examples** (Pogo Pickup 3 = pickup 2 +1, Pogo
Butterfly 4, Pogo Barfly 5, the pogo-paradox-X set, Pogo Voodoo 6). A clean ruling-vs-corpus
contradiction; either answer is internally consistent and promotes the whole family.

**Ruling options.** (A) +0: Red doctrine stands, fb.org counts normalize down. (B) +1: published
counts canonical, pogo becomes a first-class +1 modifier.

**Affected (live ~24).** pogo + pogo-{barfly, pickup, voodoo, paradox-barrage, paradox-blender,
paradox-da-da-curve, paradox-drifter, paradox-eggbeater, paradox-mirage, paradox-torque,
paradox-whirl, paradox-whirling-swirl} + variants.

### A4. Blurry follow-up  (~7 direct)
**Question.** Confirm the pending rotational treatment: is blurry +1, +2 (stepping+paradox), or
base-type-dependent?

**Evidence.** Registered flat +1; pt11 migrated the rotational bonus to +1 and **logged the
pt12 follow-up that was never delivered**: "Blurry Whirl breaks under +1, Blurry Torque breaks
under +1." Asserted ADDs split exactly: blurry = +1 on non-rotational bases (Ripwalk, Blurriest),
+2 on rotational bases (Blur 4, Blurry Whirl 5, Blurry Torque 6, Food Processor 6). Same shape
as the (now-settled) atomic rotational question.

**Ruling options.** (A) +2 uniformly (stepping+paradox): re-read Ripwalk/Blurriest as
stepping-only. (B) +1, rotational compounds stored via explicit stepping+paradox. (C) Formal
base-type-dependent split (+1 / +2 rotational).

**Affected (live ~7 direct; ripples to the modifier value dictionary-wide).** blur, blurry-whirl,
blurry-torque, blurrier, blurry-drifter, blurry-symposium-whirl, blurry-whirling-swirl.

### A5. Atomic-rotational / X-Dex  (~15-25 tricks)
**Question.** Two linked, near-settled items needing a greenlight, not fresh adjudication:
(1) **Ship the atomic = +1 migration.** atomic = +1 is FINAL in doctrine, but the live registry
still encodes atomic +1/+2-rotational; the audit is done ("13 totals move, 32 re-attributions").
(2) **X-Dex eligibility (E1):** beyond mirage (atom-smasher), are whirl / swirl X-Dex-eligible,
and does the X-Dex survive an inserted operator (the nuclear-ducking-mirage case)?

**Evidence.** All non-rotational atomic compounds are already settled +1 and active (atomic-
butterfly, atomic-osis=flux, atomic-clipper, ...). The held band is atomic on rotational bases
(mirage/whirl/swirl/torque/dyno) plus quantum/sailing/nuclear on X-Dex receivers. sumo
(nuclear-mirage) carries an explicit X-Dex; nuclear-illusion does not, so the rule is not yet
generalized.

**Ruling options.** (1) Greenlight the +1 migration (propagate the registry/parser change).
(2) State the X-Dex receiver set and whether an inserted operator (e.g. ducking) breaks it.

**Affected (live ~15-25 + 45 in-place corrections from the migration).** atomic-{whirl, swirl,
dyno, gyro-torque}, atomic-ducking-{mirage, whirl}, atomic-symposium-whirl, atomic-gyro-
symposium-whirl, sailing-illusion, sailing-symposium-mirage, nuclear-ducking-mirage, the railing
X-Dex pair.
