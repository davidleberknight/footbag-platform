## Why deferred (named-set cohort)
Railing is one of three named-set operators whose `+N` ADD comes from a set definition, not bracket tokens: **railing (+2), surfing, furious (+2)**. The content layer flags set-prefix JOB tokenization for this cohort as pending.

Writing railing alone would force a one-off verification path (`2 + brackets` plus a `(railing set) >>` prefix check) that the standard bracket-ADD invariant cannot express. That is a new verification class for a single operator. Per the decision, no separate ADD-verification path is created solely for railing.
